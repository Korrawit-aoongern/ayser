# Research Requirements
Ayser – Website & Service Health Advisor

## 1. Problem Background

ปัจจุบันผู้ดูแลเว็บไซต์หรือบริการขนาดเล็กถึงกลาง
มักไม่มีเครื่องมือที่เหมาะสมในการตรวจสอบสุขภาพระบบแบบง่าย
เครื่องมือที่มีอยู่มักซับซ้อน ต้องตั้งค่าเยอะ และต้องมีความรู้เชิง DevOps

Ayser ถูกเสนอขึ้นเพื่อแก้ปัญหานี้
โดยมุ่งเน้นการให้ข้อมูลสุขภาพระบบที่เข้าใจง่าย
และลดภาระด้านเทคนิคสำหรับผู้ใช้งาน

## 2. Target Users

จากการวิเคราะห์ persona ผู้ใช้งานหลักของระบบ ได้แก่:

- ผู้ดูแลเว็บไซต์หรือบริการขนาดเล็กที่ไม่มีความรู้ DevOps เชิงลึก
- ผู้ดูแลระบบหลายบริการที่มีเวลาจำกัด
- ผู้ใช้งานที่ต้องการทราบสถานะระบบโดยไม่ต้องตีความ metric จำนวนมาก

## 3. Research Objectives

การวิจัยนี้มีวัตถุประสงค์เพื่อ:

- ศึกษาวิธีการตรวจสอบสุขภาพระบบที่เหมาะสมกับผู้ใช้ที่ไม่ใช่ DevOps
- เปรียบเทียบวิธีการเก็บข้อมูลสุขภาพระบบในระดับต่างๆ
- ออกแบบระบบแนะนำสถานะสุขภาพที่เข้าใจง่าย
- ลดความซับซ้อนของการ monitoring โดยยังคงข้อมูลที่จำเป็น

## 4. Research Questions

คำถามหลักของการวิจัย ได้แก่:

- ผู้ใช้งานต้องการข้อมูลสุขภาพระบบในระดับใดจึงจะเพียงพอ
- วิธีการเก็บข้อมูลแบบใดให้สมดุลระหว่างความง่ายและความแม่นยำ
- จะสรุปสถานะระบบจาก metric หลายตัวให้เข้าใจง่ายได้อย่างไร
- การนำเสนอผลลัพธ์แบบใดช่วยให้ผู้ใช้งานตัดสินใจได้เร็วขึ้น

## 5. Scope of Research

ขอบเขตของงานวิจัยครอบคลุม:

- การตรวจสอบสุขภาพเว็บไซต์และบริการออนไลน์
- การเก็บข้อมูลจาก URL, Metrics Endpoint และ Cloud API
- การนำเสนอข้อมูลในระดับสรุป ไม่เน้นการวิเคราะห์เชิงลึก

ไม่ครอบคลุม:

- ระบบ monitoring เชิงลึกระดับองค์กร
- การจัดการ incident หรือ automation ขั้นสูง
- การปรับแต่ง metric เชิงเทคนิคเฉพาะทาง

## 6. Expected Outcomes

ผลลัพธ์ที่คาดหวังจากงานวิจัย ได้แก่:

- แนวทางการออกแบบระบบ monitoring ที่เรียบง่าย
- ระบบต้นแบบ (prototype) ของ Ayser
- ข้อค้นพบเกี่ยวกับระดับข้อมูลที่ผู้ใช้งานเข้าใจและใช้งานได้จริง
